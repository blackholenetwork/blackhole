# Project Checklist Templates

## Overview

This directory contains practical, ready-to-use checklist templates for each cycle of the software development process. These templates are designed to be printed or copied into your project management tools.

## Available Checklists

### 0. Master Project Checklist
**File:** `00_master_project_checklist.md`
- Complete project overview across all cycles
- High-level tracking for project managers
- Executive dashboard view
- Project closure activities

### 1. Cycle 1: Discovery & Vision Checklist
**File:** `01_cycle1_discovery_vision_checklist.md`
- Business discovery activities
- Technical assessment tasks
- Team formation steps
- 13 detailed checklist items with instructions

### 2. Cycle 2: Foundation & Design Checklist
**File:** `02_cycle2_foundation_design_checklist.md`
- Architecture design tasks
- Development practice setup
- Initial backlog creation
- 13 detailed checklist items with instructions

### 3. Cycle 3: Development Iteration Checklist
**File:** `03_cycle3_development_iteration_checklist.md`
- Sprint planning activities
- Daily development tasks
- Sprint closure ceremonies
- Reusable for each sprint

### 4. Cycle 4: Release & Operations Checklist
**File:** `04_cycle4_release_operations_checklist.md`
- Release preparation tasks
- Deployment procedures
- Operations setup
- Ongoing maintenance activities

## How to Use These Checklists

### For New Projects
1. Start with the **Master Project Checklist** for overall tracking
2. Use **Cycle 1 Checklist** when beginning project discovery
3. Progress through each cycle checklist in order
4. Keep the **Development Iteration Checklist** handy for each sprint

### For Existing Projects
1. Identify which cycle you're currently in
2. Use the appropriate cycle checklist
3. Refer to the Master Checklist for overall progress

### Customization Tips
- Add company-specific requirements
- Adjust time estimates based on team size
- Include additional approval gates as needed
- Add links to your tools and systems

## Key Features of Each Checklist

### Clear Structure
- **Objective**: Why this task matters
- **Responsible**: Primary role accountable
- **Participants**: Who needs to be involved
- **Time Estimate**: Typical duration

### Detailed Instructions
- Step-by-step actions
- Expected deliverables
- Space for notes and issues
- Completion checkboxes

### Role Clarity
Every task clearly identifies:
- Who is responsible
- Who participates
- Who approves

## Best Practices

### Daily Use
- Review relevant checklist at start of day
- Update progress throughout the day
- Note blockers and issues immediately
- Share updates in daily standup

### Sprint/Cycle Planning
- Review upcoming checklist items
- Ensure all participants are available
- Prepare required materials in advance
- Set realistic time expectations

### Continuous Improvement
- Add new items discovered during retrospectives
- Remove items that don't add value
- Update time estimates based on experience
- Share improvements with other teams

For physical copies:

- Laminate frequently used checklists
- Keep in project war room
